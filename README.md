# ⛳ GolfLink (골프링크) - MVP 제작 및 시장 검증 프로젝트

> "스크린 골프 매칭 및 레슨 중개"라는 아이디어를 **신속하게 검증**하기 위해 제작한 MVP입니다.

이 프로젝트는 '코딩' 자체가 아닌, **'아이디어의 사업 타당성 검증'**을 최우선 목표로 했습니다.
이를 위해 생성형 AI를 적극 활용하여 개발을 최소화하고, **빠른 배포**에 집중했습니다.

## 🚀 1. 나의 핵심 기여 (My Contribution)

아이디어를 실체화하고 검증하는 전 과정에 기여했습니다.

* **[기획]**
    * '시장 수요 검증'이라는 MVP 목표에 맞춰 빠른 개발을 진행하였습니다.
* **[시스템 설계]**
    * 정의된 요구사항을 바탕으로 DB 스키마와 백엔드 아키텍처를 설계했습니다.
* **[신속한 프로토타입 제작]**
    * AI를 활용해 React 컴포넌트 및 기능이 복잡한 백엔드 코드를 생성하고, 기획 의도에 맞게 디버깅하며 프로토타입을 완성시켰습니다.
* **[배포 및 운영]**
    * 신속한 배포(Vercel)와 안정적 운영(AWS EC2/RDS)을 조합한 하이브리드 인프라를 구축하고 전체 배포 과정을 총괄했습니다.
* **[마케팅/홍보 아이디어 제안]**
    * 초기 사용자 피드백 수집을 위한 마케팅 아이디어를 제안했습니다.

## 📈 2. 성과 및 핵심 배운 점 (Results & Learnings)

[성과]
> 1주일간 총 2명의 유저를 확보 했습니다.
> 이를 통해 실 수요를 검증할 수 있었고, 현재 서비스 모델의 문제점을 파악해 앞으로의 잠재적 낭비를 막게 되었습니다.


[배운점]
1.  **서비스 아이템의 명확성:** 매칭, 레슨 두 가지 아이템을 한 번에 담다 보니, 서비스의 핵심 메시지가 복잡해져 사용자의 이목을 끌지 못했습니다.
2.  **속도와 신뢰의 균형점:** '신속 배포'를 위해 개인정보를 그대로 입력받게 한 설계가 사용자의 **'심리적 저항'**을 높여 신뢰를 얻지 못했습니다.
3.  **MVP의 진정한 의미:** MVP는 '성공'을 위해서가 아니라, '실패'하더라도 **'가장 적은 비용으로 가장 빨리 실패하고 배우기 위해'** 존재함을 깨달았습니다.
